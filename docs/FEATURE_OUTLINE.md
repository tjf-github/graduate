# 云盘系统后续功能大纲（前后端对应）

> 目标：在不大改现有 C++ socket 架构的前提下，补齐“可答辩、可演示、可扩展”的核心能力。

## 1. 用户信息中心（Profile Center）

### 1.1 后端能力
- [ ] `GET /api/user/profile`：返回完整资料（用户名、邮箱、注册时间、最近登录、头像 URL、签名）
- [ ] `PUT /api/user/profile`：修改昵称/邮箱/签名
- [ ] `PUT /api/user/password`：修改密码（旧密码校验 + 新密码强度）
- [ ] `GET /api/user/security/sessions`：查看登录设备/会话列表
- [ ] `DELETE /api/user/security/sessions/{session_id}`：踢下线指定设备

### 1.2 前端页面
- [ ] 新增“个人中心”页签
- [ ] 资料卡片：头像、昵称、邮箱、注册时间、空间占用
- [ ] 安全设置：改密表单、登录设备管理
- [ ] 操作反馈：统一 toast 提示（成功/失败/处理中）

### 1.3 数据结构建议
- `users`：新增 `avatar_url`, `bio`, `last_login_at`
- `user_sessions`：记录 `device_info`, `ip`, `created_at`, `last_active_at`

## 2. 文件分享与“下载他人分享”

### 2.1 后端能力
- [ ] `POST /api/share/create`：创建分享链接（可选提取码、过期时间、访问次数）
- [ ] `GET /api/share/{share_code}`：查看分享信息（文件名、大小、失效状态）
- [ ] `POST /api/share/{share_code}/verify`：校验提取码
- [ ] `GET /api/share/{share_code}/download`：下载分享文件（无需登录或弱登录）
- [ ] `POST /api/share/{share_code}/save`：保存到“我的云盘”
- [ ] `DELETE /api/share/{share_id}`：分享者取消分享
- [ ] `GET /api/share/my`：我的分享列表（状态、浏览数、下载数）

### 2.2 前端页面
- [ ] 文件行操作里新增“分享”按钮
- [ ] 弹窗：设置有效期、提取码、权限（仅查看/可下载）
- [ ] “分享管理”页：复制链接、查看访问统计、取消分享
- [ ] “输入分享链接”页：可预览元信息并下载/转存

### 2.3 安全与风控要点
- [ ] 分享码随机且不可预测
- [ ] 下载接口限流（IP + share_code）
- [ ] 提取码错误次数阈值 + 短时封禁
- [ ] 过期后自动失效

### 2.4 数据结构建议
- `file_shares`：`id`, `owner_user_id`, `file_id`, `share_code`, `extract_code_hash`, `expires_at`, `max_downloads`, `download_count`, `status`
- `share_access_logs`：`share_id`, `ip`, `ua`, `action`, `created_at`
