# 🚀 服务器初始化菜单指南

**脚本位置**: `init.sh`  
**使用场景**: 在服务器上初始化 CloudDisk 服务  
**执行权限**: 需要 sudo 权限

---

## 📋 完整菜单选项说明

### **选项 1: 检查系统依赖** ✅
**用途**: 验证服务器环境是否满足要求

**检查内容**:
- ✓ Docker 是否已安装及版本
- ✓ Docker Compose 是否已安装及版本
- ✓ Git 是否已安装（可选）
- ✓ 项目目录是否存在
- ✓ Git 仓库是否已初始化

---

### **选项 6: 完整初始化流程** ⚡ **【推荐首次使用】**
**用途**: 一键完整初始化（自动执行 1-5 步）

**执行顺序**:
1. ✓ 检查系统依赖
2. ✓ 生成环境配置
3. ✓ 初始化 Docker 容器
4. ✓ 验证数据库
5. ✓ 运行健康检查

**耗时**: 15-20 分钟

---

### **选项 7: 快速启动** 🚀
**用途**: 只启动已停止的容器（不重建）

**何时运行**:
- 服务器重启后
- 手动停止后想快速启动

---

### **选项 8: 停止服务** ⏹️
**用途**: 停止所有容器（保留数据）

**特点**:
- ✅ 数据库数据保留
- ✅ 上传的文件保留
- ✅ 可随时启动恢复

---

### **选项 9: 查看日志** 📊
**用途**: 实时查看应用运行日志

**退出方式**: 按 `Ctrl+C`

---

### **选项 10: 清理所有数据** 🗑️ **【谨慎使用】**
**用途**: 完全删除所有容器和数据

**⚠️ 警告**: **此操作不可逆**

---

## 🎯 典型使用场景

### **场景 A: 首次服务器部署**
```bash
cd /opt/cloudisk
sudo bash init.sh
# 选择 6（完整初始化流程）
# 等待 15-20 分钟...
```

---

### **场景 B: 服务停止后重启**
```bash
cd /opt/cloudisk
sudo bash init.sh
# 选择 7（快速启动）
```

---
